## 1. 用 Hooks 解决 Skills 自动触发不可靠的问题
- **来源**: [Aakash Gupta](https://x.com/aakashgupta/status/2040888715003019433) on X | 🔥 高互动讨论帖
- **摘要**: Claude Code 的 Skills 功能有时不会根据触发词自动调用。解决方案是使用 `user_prompt_submit` Hook，编写脚本对用户输入做关键词匹配，自动将匹配到的 Skill 引用传递给 Claude，确保技能每次都能可靠触发。
- **代码示例**:
```json
// .claude/settings.json
{
  "hooks": {
    "user_prompt_submit": [
      {
        "command": "python3 .claude/match_skills.py \"$PROMPT\""
      }
    ]
  }
}
```

## 4. CLAUDE.md 保持精简 + 分层组织
- **来源**: [shanraisshan/claude-code-best-practice](https://github.com/shanraisshan/claude-code-best-practice) on GitHub | ⭐ 22K+ Stars
- **摘要**: 社区最佳实践建议：根级 CLAUDE.md 控制在 200 行以内，使用子目录中的分层 CLAUDE.md 文件管理模块级别的指令。反复出现的错误才加规则，确定性必须执行的操作用 Hooks 而非 CLAUDE.md。Skills 比 CLAUDE.md 更节省 Token，因为 Skills 按需加载，而 CLAUDE.md 每次会话都会全量加载。
- **代码示例**:
```
项目结构示例：
├── CLAUDE.md              # 全局规则（< 200 行）
├── src/
│   ├── CLAUDE.md          # 前端模块指令
│   └── api/
│       └── CLAUDE.md      # API 模块指令
└── .claude/
    └── skills/
        └── deploy.md      # 按需加载的技能
```

## 6. 用 Memory Bank 系统替代单一 CLAUDE.md 实现跨会话记忆
- **来源**: [centminmod/my-claude-code-setup](https://github.com/centminmod/my-claude-code-setup) on GitHub | ⭐ 社区推荐模板
- **摘要**: 将 CLAUDE.md 扩展为一套 Memory Bank 文件体系，包含项目概述、架构决策、开发进度等分类文件。每次会话结束时更新相关文件，下次会话时 Claude 能快速恢复上下文，解决长期项目中上下文丢失的问题。

## 7. Claude Code "穴居人模式" — 节省 75% 以上用量
- **来源**: [nathanonn.com](https://www.nathanonn.com/claude-code-caveman-mode/) | 🔥 Reddit 社区高赞分享
- **摘要**: "穴居人模式"指用极简、直白的提示词（如同穴居人说话）与 Claude Code 交互，避免冗长的自然语言描述。核心原则：短指令、明确目标、减少来回对话。实测可节省 75% 以上的 Token 用量，特别适合 API 付费用户控制成本。

## 8. SKILL.md 编写技巧 — 控制在 500 行内 + 丰富触发词
- **来源**: [ykdojo/claude-code-tips](https://github.com/ykdojo/claude-code-tips) on GitHub | ⭐ 45 条精选技巧合集
- **摘要**: Skills 文件应控制在 500 行以内，描述部分要写丰富的触发词（Claude 通过语义匹配描述来决定何时激活技能）。将常用流程封装为 Skill 比写在 CLAUDE.md 中更高效，因为 Skill 按需加载不占用每次会话的上下文窗口。
- **代码示例**:
```markdown
---
description: |
  Use when deploying, shipping, releasing,
  pushing to production, or publishing a new version.
  Triggers: deploy, ship, release, publish, push to prod
---
# Deploy Skill
1. Run tests: `npm test`
2. Build: `npm run build`
3. Deploy: `./scripts/deploy.sh`
```

## 9. TDD Guard Hook — 用 Hook 强制测试驱动开发
- **来源**: [hesreallyhim/awesome-claude-code](https://github.com/hesreallyhim/awesome-claude-code) on GitHub | ⭐ 39.9K Stars
- **摘要**: 社区开发的 TDD Guard Hook 可以在 Claude Code 修改代码时自动检查是否遵循测试驱动开发原则。如果检测到实现代码变更但没有对应的测试变更，Hook 会阻止提交并提示先编写测试。适合对代码质量有严格要求的团队。
- **代码示例**:
```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "python3 .claude/hooks/tdd_guard.py",
            "timeout": 5
          }
        ]
      }
    ]
  }
}
```

## 10. Skill 智能评分触发系统
- **来源**: [ChrisWiles/claude-code-showcase](https://github.com/ChrisWiles/claude-code-showcase) on GitHub | ⭐ 5.8K Stars
- **摘要**: 一种 Skill 智能匹配机制，通过对用户输入进行多维度评分来决定激活哪个技能：关键词匹配得 2 分、文件路径模式匹配得 4 分、意图识别匹配得 4 分。相比简单的关键词触发更精确，可以避免错误激活不相关的 Skill，特别适合项目中配置了大量 Skill 的场景。

## 12. Dippy — 智能命令权限自动审批
- **来源**: [hesreallyhim/awesome-claude-code](https://github.com/hesreallyhim/awesome-claude-code) on GitHub | ⭐ 39.9K Stars
- **摘要**: Dippy 是社区开发的 Claude Code 权限管理工具，能自动批准安全的只读命令（如 `ls`、`cat`、`git status`），同时对破坏性操作（如 `rm`、`git push --force`）弹出确认提示。解决了 Claude Code 频繁弹出权限确认框影响工作流的痛点，在保证安全的前提下大幅提升交互效率。

## 13. /loop 命令实现定期自动巡检
- **来源**: [XDA Developers](https://www.xda-developers.com/claude-codes-creator-keeps-sharing-tips-and-they-all-made-my-experience-better/) | 🔥 科技媒体报道
- **摘要**: `/loop` 命令可以让 Claude Code 在本地定期执行指定任务，最长持续 3 天。Claude Code 团队自己使用的场景包括：每 5 分钟自动处理 PR 评审和 rebase（`/loop 5m /babysit`），每 30 分钟自动整理 Slack 反馈并创建 PR（`/loop 30m /slack-feedback`）。非常适合需要持续监控和自动响应的 DevOps 场景。
- **代码示例**:
```bash
# 每 5 分钟自动处理 PR 评审、rebase、推进合并
/loop 5m /babysit

# 每 30 分钟收集 Slack 反馈并创建 PR
/loop 30m /slack-feedback

# 每 10 分钟检查 CI 状态
/loop 10m /check-ci
```
