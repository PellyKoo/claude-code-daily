# Claude Code 2.1.116 社区实用技巧 - 2026-04-21

## 1. 上下文窗口保持在 40% 以下
- **来源**: [shanraisshan/claude-code-best-practice](https://github.com/shanraisshan/claude-code-best-practice) on GitHub | ⭐ 47.4k stars
- **摘要**: 经验丰富的用户建议将上下文使用量控制在 40% 以下（最佳目标为 30% 以下）。超过 300-400k tokens 后，Claude 的智能输出质量会明显下降。遇到失败时，用双击 Esc 或 `/rewind` 回退，而不是在上下文中堆积修正指令。
- **代码示例**:
```bash
# 手动压缩并附带提示，比自动压缩效果更好
/compact 保留架构决策和测试结果，丢弃探索过程
```

## 2. 用 `.claude/rules/` 目录拆分 CLAUDE.md
- **来源**: [shanraisshan/claude-code-best-practice](https://github.com/shanraisshan/claude-code-best-practice) on GitHub | ⭐ 47.4k stars
- **摘要**: 单个 CLAUDE.md 建议不超过 200 行（有专家建议控制在 60 行以内）。将规则拆分到 `.claude/rules/` 子目录中，利用 `paths:` YAML frontmatter 实现按路径懒加载，仅在相关文件被访问时才加载对应规则。
- **代码示例**:
```yaml
# .claude/rules/frontend.md
---
paths:
  - "src/components/**"
  - "src/pages/**"
---
使用 React Server Components，避免客户端 useState 处理可在服务端获取的数据。
```

## 3. Hooks 实现自动化质量门禁
- **来源**: [ChrisWiles/claude-code-showcase](https://github.com/ChrisWiles/claude-code-showcase) on GitHub | ⭐ 5.8k stars
- **摘要**: 通过 settings.json 中的事件钩子实现自动化质量控制：`PreToolUse` 阻止在 main 分支上编辑，`PostToolUse` 自动格式化代码并运行测试，`UserPromptSubmit` 触发技能评估系统，自动匹配关键词和文件路径推荐相关技能。
- **代码示例**:
```json
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Edit|Write",
      "command": "bash -c '[[ $(git branch --show-current) != \"main\" ]]'"
    }],
    "PostToolUse": [{
      "matcher": "Edit|Write",
      "command": "npx prettier --write $CLAUDE_FILE_PATH"
    }]
  }
}
```

## 4. Dippy：用 AST 解析自动批准安全命令
- **来源**: [ldayton/Dippy](https://github.com/ldayton/Dippy) via [awesome-claude-code](https://github.com/hesreallyhim/awesome-claude-code) | ⭐ 40.3k stars (awesome 合集)
- **摘要**: Dippy 通过 AST（抽象语法树）解析 bash 命令，自动批准安全的只读操作（如 `ls`、`cat`、`grep`），同时对破坏性操作弹出确认提示。大幅减少权限疲劳，让开发者专注于真正需要审核的操作。

## 5. 用子代理隔离研究任务，保持主上下文清洁
- **来源**: [shanraisshan/claude-code-best-practice](https://github.com/shanraisshan/claude-code-best-practice) on GitHub | ⭐ 47.4k stars
- **摘要**: 将中间研究任务交给子代理（subagent）处理，避免污染主会话上下文。新任务开启新会话，相关任务可复用上下文。这是保持长会话智能输出质量的关键策略。
- **代码示例**:
```
# 在 Claude Code 中使用子代理进行隔离研究
> 帮我调研一下 Next.js App Router 的最佳缓存策略，用子代理完成，把结论告诉我

# Claude 会自动启动 Explore 子代理进行调研，主上下文不受污染
```

## 6. 用 `<important if="...">` 标签防止长指令被忽略
- **来源**: [shanraisshan/claude-code-best-practice](https://github.com/shanraisshan/claude-code-best-practice) on GitHub | ⭐ 47.4k stars
- **摘要**: 当 CLAUDE.md 较长时，Claude 可能会忽略部分规则。将关键领域规则用 `<important if="...">` 标签包裹，确保在特定条件下这些规则获得更高优先级。
- **代码示例**:
```markdown
<important if="editing database migrations">
所有迁移必须可回滚。禁止使用 DROP COLUMN，改用软删除标记。
</important>
```

## 7. ccxray：透明代理监控 Claude Code API 调用
- **来源**: [lis186/ccxray](https://github.com/lis186/ccxray) via [awesome-claude-code](https://github.com/hesreallyhim/awesome-claude-code) | ⭐ 40.3k stars (awesome 合集)
- **摘要**: ccxray 是一个透明 HTTP 代理仪表盘，捕获 Claude Code 与 Anthropic API 之间的每个请求和响应。帮助开发者调试提示词、分析 token 消耗、理解 Claude Code 内部工作原理，是优化成本和性能的利器。

## 8. 原型迭代优于冗长 PRD 文档
- **来源**: [shanraisshan/claude-code-best-practice](https://github.com/shanraisshan/claude-code-best-practice) on GitHub | ⭐ 47.4k stars
- **摘要**: 与其花时间写长篇产品需求文档（PRD），不如直接用 Claude Code 进行 20-30 轮快速原型迭代。AI 辅助编码的构建成本极低，通过实际构建来验证想法比纸上规划更高效。始终在 Plan Mode 中开始，让一个 Claude 起草方案，再让另一个"以高级工程师身份"审查。
