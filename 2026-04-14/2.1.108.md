## 技巧 1：使用 /recap 快速恢复会话上下文

**来源：** [Claude Code 官方更新日志](https://code.claude.com/docs/en/changelog) | [GitHub Releases](https://github.com/anthropics/claude-code/releases)

### 摘要

新增的 recap 功能可以在你离开并返回会话时自动提供上下文摘要，帮助你快速回忆之前的工作内容。

### 关键要点

- 在 `/config` 中可以配置 recap 功能的开关
- 使用 `/recap` 命令可以手动触发上下文回顾
- 如果禁用了遥测，可以通过设置 `CLAUDE_CODE_ENABLE_AWAY_SUMMARY` 环境变量强制启用

```bash
# 手动触发会话回顾
/recap

# 如果禁用了遥测，通过环境变量启用
export CLAUDE_CODE_ENABLE_AWAY_SUMMARY=true
```

### 实用建议

适合以下场景：午饭后回来继续编码、次日早上恢复昨天的会话、在多个项目间切换后回到之前的会话。

## 技巧 2：配置提示缓存 TTL 降低成本

**来源：** [Claude Code 官方更新日志](https://code.claude.com/docs/en/changelog) | [GitHub Issue #46829](https://github.com/anthropics/claude-code/issues/46829) | [The Register](https://www.theregister.com/2026/04/13/claude_code_cache_confusion/)

### 摘要

v2.1.108 新增了两个环境变量，让用户可以精确控制提示缓存的 TTL（存活时间），解决了此前缓存 TTL 从 1 小时静默降至 5 分钟导致的成本上涨问题。

### 关键要点

- `ENABLE_PROMPT_CACHING_1H`：选择 1 小时缓存 TTL（适用于 API key、Bedrock、Vertex 和 Foundry）
- `FORCE_PROMPT_CACHING_5M`：强制使用 5 分钟缓存 TTL
- 修复了禁用遥测的订阅用户意外回退到 5 分钟 TTL 的 bug
- 启动时如果通过 `DISABLE_PROMPT_CACHING*` 禁用了缓存，会显示警告

```bash
# 启用 1 小时缓存 TTL（推荐，可降低成本）
export ENABLE_PROMPT_CACHING_1H=true

# 强制 5 分钟缓存 TTL
export FORCE_PROMPT_CACHING_5M=true
```

### 实用建议

如果你使用 API key 或云平台（Bedrock/Vertex/Foundry），强烈建议设置 `ENABLE_PROMPT_CACHING_1H=true`，可以显著减少重复请求的成本。

## 技巧 3：内置斜杠命令自动发现

**来源：** [Claude Code 官方更新日志](https://code.claude.com/docs/en/changelog)

### 摘要

Claude 模型现在可以自动发现并调用内置的斜杠命令（如 `/init`、`/review`、`/security-review`），无需用户手动输入。

### 关键要点

- Claude 可以通过 Skill 工具自动发现和执行 `/init`、`/review`、`/security-review` 等内置命令
- 你可以在对话中直接要求 Claude 执行这些操作，它会自动选择正确的命令
- 提高了工作流的自动化程度

### 实用建议

无需记忆所有可用的斜杠命令，直接用自然语言描述你的需求即可。例如，你可以说"帮我审查这个 PR"，Claude 会自动调用 `/review`。

## 技巧 4：/undo 命令快速回退

**来源：** [Claude Code 官方更新日志](https://code.claude.com/docs/en/changelog) | [Claude Code 命令速查表](https://computingforgeeks.com/claude-code-cheat-sheet/)

### 摘要

`/undo` 现在是 `/rewind` 的别名，更符合直觉的命令名称让回退操作更容易记忆。

### 关键要点

- `/undo` 和 `/rewind` 功能完全相同
- 打开检查点菜单，可以回退所有文件更改同时保留对话历史
- 双击 `Escape` 键也能快速打开回退菜单
- 当 Claude 的方法错误但分析正确时，这个功能特别有用

```bash
# 以下两个命令效果相同
/undo
/rewind

# 快捷键：双击 Escape 也能打开回退菜单
```

## 技巧 5：改进的 /resume 会话选择器

**来源：** [Claude Code 官方更新日志](https://code.claude.com/docs/en/changelog)

### 摘要

`/resume` 命令的会话选择器现在默认只显示当前目录的会话，减少了在多项目间的干扰。

### 关键要点

- 默认过滤为当前项目目录的会话
- 按 `Ctrl+A` 可以显示所有项目的会话
- 配合 `--continue` 标志可自动恢复最近的会话
- 配合 `--resume` 可以从交互列表中选择特定会话

```bash
# 在 CLI 中恢复最近的会话
claude --continue

# 从列表中选择特定会话
claude --resume

# 恢复指定会话 ID
claude --resume <session-id>

# 在交互模式中使用
/resume
# 然后按 Ctrl+A 查看所有项目的会话
```

## 技巧 6：更好的错误信息区分

**来源：** [Claude Code 官方更新日志](https://code.claude.com/docs/en/changelog)

### 摘要

v2.1.108 改进了错误信息展示，现在可以清楚区分服务器速率限制和计划用量限制，并在 5xx/529 错误时提供 status.claude.com 链接。

### 关键要点

- 服务器速率限制（rate limit）和计划用量限制（plan usage limit）现在有明确区分
- 5xx 和 529 错误会显示 [status.claude.com](https://status.claude.com) 链接，方便检查服务状态
- 输入不存在的斜杠命令时，会建议最接近的匹配命令
- `/model` 命令在切换模型时会发出警告，因为下次响应会重新读取未缓存的完整历史

### 实用建议

遇到错误时注意区分是速率限制还是用量限制：速率限制通常等几秒即可恢复，而用量限制可能需要等待更长的重置周期或升级计划。

## 技巧 8：Routines（自动化例程）- 全新自动化工作流

**来源：** [Anthropic 官方博客](https://claude.com/blog/introducing-routines-in-claude-code) | [Claude Code 文档](https://code.claude.com/docs/en/routines) | [9to5Mac](https://9to5mac.com/2026/04/14/anthropic-adds-repeatable-routines-feature-to-claude-code-heres-how-it-works/) | [SiliconANGLE](https://siliconangle.com/2026/04/14/anthropics-claude-code-gets-automated-routines-desktop-makeover/)

### 摘要

Anthropic 于 4 月 14 日宣布推出 Routines（研究预览版），这是一种在 Anthropic 云基础设施上运行的自动化 Claude Code 配置，支持定时触发、API 调用触发和 GitHub 事件触发。

### 关键要点

- **Routine 是什么**：一个保存好的 Claude Code 配置，包含提示词、仓库和连接器，可自动运行
- **三种触发方式**：
  - **定时触发**（Schedule）：每小时、每晚、每周等周期运行
  - **API 触发**：通过 HTTP POST 请求按需触发
  - **GitHub 事件触发**：响应 PR、推送、Issue 等仓库事件
- **无需开着电脑**：Routine 在 Anthropic 托管的云基础设施上运行
- **适用计划**：Pro、Max、Team、Enterprise

### 创建 Routine

```bash
# 在 CLI 中使用 /schedule 创建定时 Routine
/schedule daily PR review at 9am

# 列出所有 Routine
/schedule list

# 更新 Routine
/schedule update

# 立即运行 Routine
/schedule run
```

也可以在网页端操作：访问 [claude.ai/code/routines](https://claude.ai/code/routines) 创建和管理。

### API 触发示例

```bash
curl -X POST https://api.anthropic.com/v1/claude_code/routines/trig_01ABCDEFGHJKLMNOPQRSTUVW/fire \
  -H "Authorization: Bearer sk-ant-oat01-xxxxx" \
  -H "anthropic-beta: experimental-cc-routine-2026-04-01" \
  -H "anthropic-version: 2023-06-01" \
  -H "Content-Type: application/json" \
  -d '{"text": "Sentry alert SEN-4521 fired in prod. Stack trace attached."}'
```

### 实用场景

| 场景 | 触发方式 | 说明 |
|------|---------|------|
| 待办事项维护 | 定时 | 每晚对 Issue 分类打标签 |
| 告警分流 | API | 监控工具触发后自动分析并提 PR |
| 代码审查 | GitHub 事件 | PR 创建时自动执行团队审查清单 |
| 部署验证 | API | CD 流水线部署后自动运行冒烟测试 |
| 文档漂移检查 | 定时 | 每周扫描合并的 PR，标记过时文档 |
| SDK 同步移植 | GitHub 事件 | 一个 SDK 合并 PR 后自动移植到另一语言版本 |

### 提示词编写建议

Routine 提示词必须自包含且明确：

```
# 不好的写法
review PRs

# 好的写法
review open PRs against the /auth module, check for SQL injection
vulnerabilities and missing input validation, leave inline comments
with severity labels, and add a summary comment with pass/fail status.
```

## 技巧 9：桌面应用全新设计 - 多会话并行工作

**来源：** [Anthropic 官方博客](https://claude.com/blog/claude-code-desktop-redesign) | [Claude Code 文档](https://code.claude.com/docs/en/desktop) | [X/Twitter @claudeai](https://x.com/claudeai/status/2044131493966909862)

### 摘要

同日发布的桌面应用全新设计支持在单个窗口中并行运行多个 Claude 会话，并新增了侧边栏管理、集成终端、文件编辑器和可拖拽布局。

### 关键要点

- **多会话并行**：在一个窗口中同时运行多个 Claude 会话，每个会话自动使用独立的 Git worktree 隔离
- **可拖拽布局**：终端、文件编辑器、预览面板均可自由拖拽排列
- **集成终端**：无需切换窗口即可使用终端
- **侧边对话**（Side Chats）：从主线程分支出来提问，不影响主会话上下文
- **可视化 Diff 审查**：内联评论的差异查看器
- **应用预览**：支持 HTML 和 PDF 预览
- **PR 监控**：自动修复、自动合并、自动归档

### 快捷键

```bash
# 新建会话
Cmd+N (macOS) / Ctrl+N (Windows)

# 在侧边栏中管理所有会话
点击 "+ New session" 或使用快捷键
```

### 实用建议

利用多会话功能并行处理不同任务：一个会话负责功能开发，另一个处理代码审查，第三个查阅文档。每个会话自动使用独立的 Git worktree，避免代码冲突。

### 摘要

v2.1.108 修复了多个影响日常使用的 bug，以下是最值得关注的修复：

### 关键修复列表

| 修复内容 | 影响 |
|---------|------|
| 修复 `/login` 中无法粘贴的问题 | 解决 v2.1.105 引入的回退 |
| 修复 `DISABLE_TELEMETRY` 用户缓存 TTL 回退到 5 分钟 | 避免不必要的成本增加 |
| 修复 `--resume <session-id>` 丢失自定义名称和颜色 | 保持会话个性化设置 |

| 修复策略管理的插件不自动更新 | 跨项目插件更新正常工作 |

### 实用建议

如果你遇到以下问题，升级到 v2.1.108 即可解决：
- 登录时无法粘贴验证码
- 使用 `--teleport` 远程连接后显示异常
- 非英语环境下字符丢失
- 长会话中内存占用过高
