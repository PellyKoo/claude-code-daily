## 3. 短会话 + 检查点策略，避免"修越多错越多"
- **来源**: [Reddit r/ClaudeAI 社区共识](https://www.reddit.com/r/ClaudeAI/) | 🔥 社区广泛认同的最佳实践
- **摘要**: 社区一致认为：短而专注的会话优于长时间马拉松。每次自主操作前先 commit 一个检查点。如果结果不对，回滚比"修复前进"成功率高得多。长会话容易导致上下文退化。
- **代码示例**:
```bash
# 每次让 Claude 执行大改动前先创建检查点
git add -A && git commit -m "checkpoint before refactor"

# 如果结果不理想，直接回滚
git reset --hard HEAD~1

# 或使用 Claude Code 内置回退
/undo
```

## 4. CLAUDE.md 精简原则 — 少即是多
- **来源**: [Boris Cherny (Claude Code 创建者)](https://x.com/bcherny/status/2007179832300581177) on X | 🔥 2.8K+ 点赞
- **摘要**: Claude Code 创建者 Boris Cherny 分享了他的工作流，出乎意料地"原味"。核心建议：CLAUDE.md 保持在 200 行以内，从零开始，只有当你发现反复对 Claude 说同样的话时才添加规则。域特定规则用 `<important if="...">` 标签包裹，防止文件变长后被忽略。
- **代码示例**:
```markdown
<!-- CLAUDE.md 示例 -->
# 项目规范
- TypeScript strict mode
- 测试覆盖率 > 80%

<important if="editing API routes">
所有 API 路由必须包含输入验证和错误处理
</important>
```

## 6. tmux 远程会话 — 飞机上也能用 Claude Code
- **来源**: [Matt Van Horn](https://x.com/mvanhorn/status/2035857346602340637) on X | 🔥 实用离线方案
- **摘要**: Claude Code 在不稳定网络下容易断连。解决方案：先 tmux 连接到你的 Mac Mini 或远程服务器，Claude Code 会话运行在服务器端，笔记本只是一个显示窗口。即使断网，会话也不会丢失。
- **代码示例**:
```bash
# 在远程服务器上创建 tmux 会话
ssh your-server
tmux new -s claude-work

# 启动 Claude Code
claude

# 断网后重新连接
ssh your-server
tmux attach -t claude-work
```

- **摘要**: Santiago 在 Sonar Summit 分享的 18 条 Claude Code 技巧中，排第一的就是：始终使用 `@filename.py` 或 `@src/classes/` 引用文件路径来约束代理的操作范围，避免 Claude 在整个代码库中"乱逛"。

```bash

# 指定目录范围
@src/components/ 将所有 class 组件迁移到函数组件
```
