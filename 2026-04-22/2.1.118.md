## 1. Git Worktrees 实现多任务并行开发
- **来源**: [Boris Cherny](https://x.com/bcherny/status/2017742741636321619) on X | 🔥 高互动（Claude Code 创始人官方分享）
- **摘要**: Claude Code 团队推荐使用 Git Worktrees 实现并行开发。每个 worktree 是独立的工作目录，可以同时运行多个 Claude Code 会话处理不同任务，互不干扰。适合同时修复 bug、开发新功能、做代码审查等场景。
- **代码示例**:
```bash
# 创建 worktree 处理不同任务
git worktree add ../feature-auth feature/auth
git worktree add ../bugfix-api bugfix/api-timeout

# 在每个 worktree 中分别启动 Claude Code 会话
cd ../feature-auth && claude
cd ../bugfix-api && claude

# 完成后清理
git worktree remove ../feature-auth
```

## 2. 用 AskUserQuestion 工具让 Claude 反向采访你
- **来源**: [Tech with Mak](https://x.com/techNmak/status/2037788648691884207) on X | 🔥 高互动转发
- **摘要**: 不要一次性写完所有需求，而是让 Claude 主动提问来澄清需求。告诉 Claude "用 AskUserQuestion 工具采访我"，它会逐步提出关键问题，帮你理清需求盲点，最终生成更精准的实现方案。
- **代码示例**:
```
# 在 Claude Code 中输入：
> 我想给项目添加用户认证功能，请用 AskUserQuestion 工具采访我，
  了解所有必要的需求细节后再开始实现。

# Claude 会依次询问：
# - 认证方式（OAuth/JWT/Session）？
# - 需要哪些第三方登录？
# - 密码策略要求？
# - 是否需要 2FA？
```

## 3. 白板星号法：Claude Code 作为团队成员管理任务
- **来源**: [Charlie Holtz](https://x.com/charliebholtz/status/1935029747815432505) on X | 🔥 高互动讨论
- **摘要**: Chorus 团队把 Claude Code 视为正式团队成员，在白板上用星号(*)标记由 Claude 主要负责的任务，星号数量持续增长。关键经验：给 Claude 清晰的验证标准（测试用例、截图对比），让它独立完成任务后人工审查，而非全程监督。
- **代码示例**:
```markdown
# 团队白板任务分配示例
- [ ] 用户仪表盘 UI 重构 *
- [ ] API 速率限制实现 *
- [ ] 产品路线图会议
- [ ] 数据库索引优化 *
- [ ] 客户反馈分析
# * = Claude Code 主要负责
```

## 4. 50 条实战技巧精选：截图调试与 MCP 集成
- **来源**: [Miles Deutscher](https://x.com/milesdeutscher/status/2014027971254952332) on X | 🔥 高互动长文
- **摘要**: 当 Claude 陷入循环或无法理解 UI 问题时，直接截图给它看比文字描述高效 10 倍。结合 MCP 服务器让 Claude 直接读取 Chrome 控制台日志，实现端到端的前端调试闭环。
- **代码示例**:
```bash
# 安装 Chrome DevTools MCP 服务器
claude mcp add chrome-devtools -- npx @anthropic-ai/chrome-devtools-mcp

# 然后在 Claude Code 中：
> 页面上的按钮点击没反应，我已经截图放在 /tmp/screenshot.png，
  同时请通过 MCP 查看 Chrome 控制台报错信息来帮我定位问题。
```

## 5. 用 /loop 实现定时巡检（最长 3 天）
- **来源**: [Boris Cherny](https://x.com/bcherny/status/2017742741636321619) on X | 🔥 Claude Code 团队官方工作流
- **摘要**: `/loop` 命令可以让 Claude Code 按固定间隔重复执行任务，最长持续 3 天。适合监控部署状态、定期运行测试套件、检查依赖更新等场景。比手动轮询或编写 cron 脚本更轻量。
- **代码示例**:
```bash
# 每 5 分钟检查一次部署状态
/loop 5m 检查生产环境部署是否完成，如果有错误立即通知我

# 每 10 分钟运行测试（默认间隔）
/loop 运行 npm test 并报告失败的用例

# 每 30 分钟检查依赖安全漏洞
/loop 30m npm audit 并总结高危漏洞
```

## 7. Telegram 集成：手机远程操控 Claude Code
- **来源**: [Matt Van Horn](https://x.com/mvanhorn/status/2035857346602340637) on X | 🔥 "Every Claude Code Hack I Know" 高互动帖
- **摘要**: 通过 Telegram Bot 集成，可以直接从手机发送指令给运行中的 Claude Code 会话。外出时发一条消息如 "/ce:plan 修复超时问题"，回到电脑前方案已经准备好。实现真正的异步编程工作流。
- **代码示例**:
```bash
# Telegram 消息示例
/ce:plan fix the timeout issue in the payment service
/ce:review check PR #42 for security issues
/ce:run npm test -- --grep "auth"

# 回到电脑时，Claude Code 已完成分析，方案等待你审阅
```

## 8. BMAD 方法论：结构化大型项目开发
- **来源**: [wesammustafa/Claude-Code-Everything-You-Need-to-Know](https://github.com/wesammustafa/Claude-Code-Everything-You-Need-to-Know) on GitHub | ⭐ 综合指南
- **摘要**: BMAD（Build, Measure, Adapt, Deploy）方法论将大型项目拆解为可控的迭代周期。每个周期包括：用 Plan Mode 构建方案、用测试验证结果、根据反馈调整、最终部署。特别适合用 Claude Code 开发复杂功能时避免上下文过载。
- **代码示例**:
```markdown
# BMAD 工作流
## Build（构建）
> /plan 设计用户权限系统的数据模型和 API

## Measure（验证）
> 运行测试套件，确保所有边界情况覆盖

## Adapt（调整）
> /compact 保留架构决策，开始新会话优化性能

## Deploy（部署）
> 创建 PR，用 /review 进行代码审查
```
