# Claude Code 2.1.111 社区实用技巧 - 2026-04-15

## 1. 在 50% 上下文时主动执行 /compact
- **来源**: [shanraisshan/claude-code-best-practice](https://github.com/shanraisshan/claude-code-best-practice) on GitHub | ⭐ 45.5k stars
- **摘要**: 当上下文窗口使用超过 50% 时，手动执行 `/compact` 触发上下文压缩。避免进入"代理变笨区"——上下文过满会显著降低推理质量。同理，当 Claude 偏离方向时，优先使用 `/rewind`（或连按两次 Esc）回退，而非在同一上下文中反复纠正。
- **代码示例**:
```
# 在 Claude Code 会话中随时执行
/compact

# 回退到之前的状态
/rewind
```

## 2. 多代理模式：Opus 规划 + Sonnet 执行
- **来源**: [r/ClaudeAI](https://www.reddit.com/r/ClaudeAI/) on Reddit + [shanraisshan/claude-code-best-practice](https://github.com/shanraisshan/claude-code-best-practice) on GitHub | ⭐ 45.5k stars
- **摘要**: Reddit 社区最热议的高级技巧——主会话使用 Opus 进行复杂推理和规划，子代理使用 Sonnet 处理聚焦任务。通过环境变量 `CLAUDE_CODE_SUBAGENT_MODEL` 控制子代理模型，在推理质量和成本之间取得平衡。
- **代码示例**:
```bash
# 设置子代理使用 Sonnet 模型
export CLAUDE_CODE_SUBAGENT_MODEL=claude-sonnet-4-6

# 主会话默认使用 Opus 进行规划
claude --model claude-opus-4-6
```

## 3. 自定义状态栏实时显示关键信息
- **来源**: [ykdojo/claude-code-tips](https://github.com/ykdojo/claude-code-tips) on GitHub | ⭐ 7.7k stars, 570 forks
- **摘要**: 通过自定义状态栏脚本，在 Claude Code 底部实时显示当前模型、Git 分支、未提交文件数、Token 使用量等信息，提供 10 种配色主题。帮助开发者随时掌握会话状态，避免意外超出上下文限制。
- **代码示例**:
```jsonc
// .claude/settings.json
{
  "statusLine": {
    "script": "echo \"$(git branch --show-current) | $(git status --short | wc -l) changed\""
  }
}
```

## 4. 用 Git Worktrees 实现并行开发
- **来源**: [Tech with Mak](https://x.com/techNmak/status/2037788648691884207) on X + [shanraisshan/claude-code-best-practice](https://github.com/shanraisshan/claude-code-best-practice) on GitHub | ⭐ 45.5k stars
- **摘要**: 利用 Git Worktrees 让多个 Claude Code 会话同时在不同分支上工作，互不干扰。一个代理修 bug，另一个加功能，第三个跑测试——真正的并行开发。搭配 tmux 可以在远程机器上运行多个持久会话。
- **代码示例**:
```bash
# 创建工作树用于并行开发
git worktree add ../feature-auth feature/auth
git worktree add ../bugfix-login bugfix/login

# 在不同终端窗口分别启动 Claude Code
cd ../feature-auth && claude
cd ../bugfix-login && claude
```

## 5. Dippy 智能命令审批钩子
- **来源**: [awesome-claude-code](https://github.com/hesreallyhim/awesome-claude-code) on GitHub | ⭐ 39.1k stars, 3.2k forks
- **摘要**: Dippy 通过 AST 语法树解析自动审批安全的 Bash 命令（如 `ls`、`cat`、`git status`），同时拦截危险操作（如 `rm -rf`、`git push --force`）并提示用户确认。大幅减少权限弹窗，同时保持安全防护。
- **代码示例**:
```jsonc
// .claude/settings.json - 配置 Dippy 钩子
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "command": "dippy check --ast \"$TOOL_INPUT\""
      }
    ]
  }
}
```

## 6. /tdd 测试驱动开发工作流命令
- **来源**: [awesome-claude-code](https://github.com/hesreallyhim/awesome-claude-code) on GitHub | ⭐ 39.1k stars, 3.2k forks
- **摘要**: 社区贡献的 `/tdd` 自定义命令，强制执行红-绿-重构（Red-Green-Refactor）纪律。先写失败的测试，再让 Claude 写实现代码使测试通过，最后重构。确保代码质量从一开始就有测试保障。
- **代码示例**:
```markdown
<!-- .claude/commands/tdd.md -->
# TDD Workflow
1. Write a failing test for: $ARGUMENTS
2. Run the test suite - confirm it FAILS
3. Write minimal code to make the test pass
4. Run the test suite - confirm it PASSES
5. Refactor if needed, keeping tests green
```

## 8. Handoff 文档模式跨会话传递上下文
- **来源**: [ykdojo/claude-code-tips](https://github.com/ykdojo/claude-code-tips) on GitHub | ⭐ 7.7k stars, 570 forks
- **摘要**: 在结束一个复杂会话前，让 Claude 生成一份 Handoff 文档，记录当前进度、已完成的工作、待解决的问题和关键决策。新会话启动时读取此文档即可无缝接续，避免重复解释背景和丢失上下文。
- **代码示例**:
```bash
# 会话结束前生成 handoff 文档
# 在 Claude Code 中输入：
"请生成一份 handoff 文档，总结当前进度、未完成的任务和关键决策"

# 新会话开始时
claude "请阅读 HANDOFF.md 并继续之前的工作"
```
