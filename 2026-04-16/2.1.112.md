## 1. /ultraplan 四代理并行规划
- **来源**: [Nick Spisak](https://x.com/NickSpisak_/status/2043060265823146202) on X | 🔥 高互动讨论
- **摘要**: Claude Code 新增 `/ultraplan` 命令，启动 4 个 Opus 代理并行规划项目——3 个探索者分头调研，1 个评审者汇总审查。相比单代理规划速度提升约 4 倍，但更重要的是评审工作流能发现单代理容易遗漏的问题。适合大型项目初始化和复杂重构场景。
- **代码示例**:
```
# 在 Claude Code 会话中直接使用
/ultraplan 重构用户认证模块，支持 OAuth2 和 SAML

# ultraplan 自动启动:
# - Explorer 1: 分析现有代码结构
# - Explorer 2: 调研依赖和 API
# - Explorer 3: 评估测试覆盖率
# - Critic: 汇总审查并生成最终方案
```

## 2. Hooks 修复 Skills 自动触发失灵
- **来源**: [Aakash Gupta](https://x.com/aakashgupta/status/2040888715003019433) on X | 🔥 高互动讨论
- **摘要**: Claude Code 的 Skills 经常无法可靠地自动触发——你写了 "use when the user mentions slides"，结果一半时间被忽略。Carl Vellotti 的解决方案：用 `UserPromptSubmit` 钩子做关键词匹配，自动将匹配到的 Skill 引用注入给 Claude，变"被动希望触发"为"主动确保触发"。
- **代码示例**:
```jsonc
// .claude/settings.json
{
  "hooks": {
    "UserPromptSubmit": [
      {
        "matcher": "",
        "command": "python3 .claude/scripts/skill-matcher.py \"$PROMPT\""
      }
    ]
  }
}
```
```python
# .claude/scripts/skill-matcher.py
# 扫描 .claude/skills/ 下所有 SKILL.md
# 对用户输入做关键词匹配
# 匹配成功则输出 skill 引用供 Claude 使用
```

## 3. claude-mem 静默钩子记忆系统
- **来源**: [Ben Sigman](https://x.com/bensig/status/2043729727148699739) on X | 🔥 AI Memory Awards 2026 最佳工作流集成
- **摘要**: 荣获 2026 年 4 月 AI Memory Awards "最佳现有工作流集成"奖。claude-mem 通过钩子静默捕获 Claude Code 会话中的关键信息，无需手动操作即可跨会话持久化记忆。本地优先，不依赖外部服务，适合注重隐私的开发者。
- **代码示例**:
```jsonc
// .claude/settings.json - 配置 claude-mem 钩子
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "",
        "command": "claude-mem capture --silent"
      }
    ]
  }
}
```
```bash
# 安装 claude-mem
npm install -g claude-mem

# 查看已捕获的记忆
claude-mem list

# 在新会话中加载记忆上下文
claude-mem recall --topic "auth refactor"
```

## 4. CLAUDE.md 安全风险警示——恶意配置可劫持工作流
- **来源**: [Unclecode (Hossein)](https://x.com/unclecode/status/2042592340611842119) on X | 🔥 大量讨论与转发
- **摘要**: 社区热议安全话题：Claude Code 对 `~/.ssh`、`~/.aws`、`.env` 有完整访问权限，且可以执行 `curl/wget`。一个恶意的 CLAUDE.md 文件理论上可以劫持整个开发工作流。建议：(1) 审查第三方仓库的 CLAUDE.md 后再运行 Claude Code；(2) 使用 hooks 限制敏感目录访问；(3) 在沙盒环境或容器中运行不受信任的项目。
- **代码示例**:
```jsonc
// .claude/settings.json - 添加安全防护钩子
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "command": "python3 .claude/scripts/security-guard.py \"$TOOL_INPUT\""
      }
    ]
  }
}
```
```bash
# 在 Docker 容器中运行不受信任的项目
docker run -it --rm -v $(pwd):/workspace -w /workspace \
  node:20 npx claude-code
```

## 5. CLAUDE.md 分层架构与 rules/ 作用域规则
- **来源**: [shanraisshan/claude-code-best-practice](https://github.com/shanraisshan/claude-code-best-practice) on GitHub | ⭐ 45.5k stars + [abhishekray07/claude-md-templates](https://github.com/abhishekray07/claude-md-templates) on GitHub
- **摘要**: 大型项目不要把所有规则塞进一个 CLAUDE.md（超过 80 行 Claude 就会开始忽略部分内容）。推荐分层策略：根目录放全局规则（<60行），子目录放局部规则。进阶用法：在 `.claude/rules/` 目录下用 YAML frontmatter 的 `globs` 字段限定规则生效范围，实现按文件类型或路径自动加载对应规则。
- **代码示例**:
```
项目结构:
├── CLAUDE.md                    # 全局规则 (<60 行)
├── src/
│   ├── auth/CLAUDE.md           # 认证模块专用规则
│   └── api/CLAUDE.md            # API 模块专用规则
└── .claude/
    └── rules/
        ├── typescript.md        # TS 文件专用规则
        └── testing.md           # 测试文件专用规则
```
```yaml
# .claude/rules/typescript.md
---
globs: "**/*.ts,**/*.tsx"
---
- 使用 strict TypeScript，禁止 any
- 所有函数必须有明确的返回类型
- 使用 zod 进行运行时类型校验
```

## 6. /btw 侧问命令——不中断当前任务提问
- **来源**: [r/ClaudeAI](https://www.reddit.com/r/ClaudeAI/) on Reddit | 🔥 Reddit 热门讨论
- **摘要**: 2026 年 3 月新增的 `/btw` 命令解决了一个常见痛点：Claude 正在执行复杂任务时，你突然想问个不相关的问题，但又不想打断它或污染上下文。`/btw` 开启一个侧问通道，拥有当前会话的完整可见性但不会干扰主任务流。问完即走，主任务继续。
- **代码示例**:
```
# Claude 正在重构数据库模块...

# 你突然想确认一个 API 格式
/btw 我们的 REST API 是用 snake_case 还是 camelCase?

# Claude 回答侧问后，自动恢复重构任务
# 主对话历史不受影响
```
