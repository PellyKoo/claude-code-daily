## 4. Git Worktree 并行开发：多分支同时作业不冲突
- **来源**: [ykdojo/claude-code-tips](https://github.com/ykdojo/claude-code-tips) on GitHub | ⭐ 7.9k stars
- **摘要**: 利用 Git worktree 在不同目录中同时处理多个分支，每个 Claude Code 实例在独立的工作树中运行，彻底消除分支切换和合并冲突。Reddit 社区推荐主会话使用 Opus 做复杂推理，子代理使用 Sonnet 处理聚焦任务，配合 worktree 实现真正的并行开发。
- **代码示例**:
```bash
# 创建独立工作树
git worktree add ../feature-auth feature/auth
git worktree add ../bugfix-api bugfix/api-timeout

# 在不同终端窗口启动 Claude Code
cd ../feature-auth && claude   # 终端 1：开发认证功能
cd ../bugfix-api && claude     # 终端 2：修复 API 超时

# 完成后清理
git worktree remove ../feature-auth
git worktree remove ../bugfix-api

# 高级模式：Opus 主控 + Sonnet 子代理
# 主会话（Opus）：架构设计和复杂逻辑
# 子代理（Sonnet）：测试编写、文档更新、代码搜索
```

## 5. Hooks 确定性自动化：比 CLAUDE.md 指令更可靠
- **来源**: [Aakash Gupta](https://x.com/aakashgupta/status/2040888715003019433) on X | 🔥 高互动
- **摘要**: CLAUDE.md 中的指令是"建议性"的，Claude 可能忽略；而 Hooks 是"确定性"的脚本，在特定事件（消息发送前、压缩后、响应返回时）自动触发，100% 执行。当 skills 不能可靠自动触发时，用 hooks 做关键词匹配并手动激活对应 skill，实现零遗漏的工作流自动化。
- **代码示例**:
```json
// .claude/settings.json - hooks 配置示例
{
  "hooks": {
    "UserPromptSubmit": [
      {
        "command": "bash .claude/hooks/skill-matcher.sh \"$PROMPT\"",
        "description": "根据用户输入自动匹配并激活 skill"
      }
    ],
    "PostToolUse": [
      {
        "command": "bash .claude/hooks/auto-format.sh",
        "description": "工具调用后自动格式化代码"
      },
      {
        "command": "bash .claude/hooks/run-tests.sh",
        "description": "代码修改后自动运行相关测试"
      }
    ]
  }
}
```

## 7. RIPER 工作流：强制分离研究、创新、计划、执行、审查五阶段
- **来源**: [hesreallyhim/awesome-claude-code](https://github.com/hesreallyhim/awesome-claude-code) on GitHub | ⭐ 41k stars
- **摘要**: RIPER 是社区设计的结构化工作流，将开发过程强制分为 Research（研究）→ Innovate（创新）→ Plan（计划）→ Execute（执行）→ Review（审查）五个阶段，防止 Claude 在理解需求前就开始写代码。每个阶段有明确的输入输出要求，确保代码质量和方向正确性。
- **代码示例**:
```markdown
# RIPER 工作流在 CLAUDE.md 中的配置

## 工作模式
当收到新功能需求时，严格按以下顺序执行：

### R - Research（研究）
- 搜索相关代码和依赖
- 理解现有架构和约束
- 输出：调研报告

### I - Innovate（创新）
- 提出 2-3 种实现方案
- 分析各方案优劣
- 输出：方案对比

### P - Plan（计划）
- 选定方案后制定详细步骤
- 识别风险点和测试策略
- 输出：实施计划

### E - Execute（执行）
- 按计划逐步实现
- 每步完成后运行测试
- 输出：可工作的代码

### R - Review（审查）
- 自我审查代码质量
- 检查边界情况和安全性
- 输出：审查报告 + 最终提交
```

# 3. Claude Code 实现代码
