## 1. Dippy — 智能自动审批安全 Bash 命令
- **来源**: [hesreallyhim/awesome-claude-code](https://github.com/hesreallyhim/awesome-claude-code) on GitHub | ⭐ 40.1k stars
- **摘要**: 使用 Claude Code 时频繁弹出权限确认提示非常烦人。Dippy 通过 AST 解析判断命令是否安全，自动批准无害的 bash 命令（如 `ls`、`cat`、`git status`），在不关闭安全防护的前提下大幅减少权限弹窗干扰。
- **代码示例**:
```json
// .claude/settings.json — hooks 配置示例
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "command": "dippy check --ast"
      }
    ]
  }
}
```

## 3. 自定义状态栏 — 实时显示模型与 Token 信息
- **来源**: [ykdojo/claude-code-tips](https://github.com/ykdojo/claude-code-tips) on GitHub | ⭐ 7.9k stars
- **摘要**: 在 Claude Code 底部添加自定义状态栏，实时显示当前模型名称、Git 分支、未提交文件数和 token 使用量。仓库提供了一个 bash 脚本，支持 10 种配色主题，帮助你随时感知上下文消耗情况，避免意外用尽额度。
- **代码示例**:
```bash
# 在 ~/.claude/settings.json 中启用状态栏
{
  "statusLine": {
    "enabled": true,
    "script": "~/.claude/status-line.sh"
  }
}
```

## 5. Docker 容器化会话 — 隔离运行高风险任务
- **来源**: [ykdojo/claude-code-tips](https://github.com/ykdojo/claude-code-tips) on GitHub | ⭐ 7.9k stars
- **摘要**: 将 Claude Code 放入 Docker 容器中运行，配合 `--dangerously-skip-permissions` 参数实现全自动化。适合运行有风险的研究性任务、系统提示词补丁或多 Agent 编排场景。容器隔离确保即使出错也不会影响宿主系统。
- **代码示例**:
```bash
docker run -it --rm \
  -v $(pwd):/workspace \
  -e ANTHROPIC_API_KEY=$ANTHROPIC_API_KEY \
  claude-code:latest \
  --dangerously-skip-permissions \
  -p "重构 src/ 下的所有测试文件"
```

## 6. 用 Hooks 修复 Skills 不可靠触发问题
- **来源**: [Aakash Gupta (@aakashgupta)](https://x.com/aakashgupta/status/2040888715003019433) on X
- **摘要**: Claude Code 的 Skills 经常不能可靠地自动触发（即使写了 "use when..." 描述）。解决方案是改用 Hooks：在 `PreToolUse` 或 `PostToolUse` 事件上绑定脚本，确定性地在特定操作前后执行自定义逻辑，而不依赖 Claude 的语义匹配判断。
- **代码示例**:
```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit",
        "command": "node .claude/hooks/enforce-style.js"
      }
    ]
  }
}
```

## 7. /ultraplan — 4 个 Opus Agent 并行规划
- **来源**: [Nick Spisak (@NickSpisak_)](https://x.com/NickSpisak_/status/2043060265823146202) on X
- **摘要**: Claude Code 新推出的 `/ultraplan` 命令用 4 个 Opus agent 并行工作：3 个 Explorer 探索代码库不同区域，1 个 Critic 审核和整合结果。相比普通 Plan 模式，不仅速度快 4 倍，更重要的是多视角的审查机制能产出更高质量的实施方案。适合复杂项目的架构规划和重构场景。

## 8. TDD Guard — 强制测试驱动开发的 Hook
- **来源**: [hesreallyhim/awesome-claude-code](https://github.com/hesreallyhim/awesome-claude-code) on GitHub | ⭐ 40.1k stars
- **摘要**: TDD Guard 是一个实时文件监控 Hook，强制执行测试驱动开发流程：Claude 必须先写测试再写实现代码。它监控文件操作顺序，如果检测到在没有对应测试的情况下修改源文件，会阻止操作并提醒先补测试。确保 AI 辅助开发也遵循 TDD 纪律。
